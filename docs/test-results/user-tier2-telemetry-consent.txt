## Focused backend controller tests

Command:

`go test ./controller -run 'TestTelemetryConsent|TestAdminUpdateUserCannotEnableTelemetryConsent'`

Dataset / fixtures / mocks:

- In-memory SQLite `users` table migrated from `model.User`.
- Test users with unique usernames and affiliate codes.
- Redis cache disabled for the test scope so model update paths do not require a Redis client.

Assertions:

- Current user can read Tier 2 telemetry consent state.
- Current user can enable consent and receives a consent timestamp.
- Current user can disable consent while retaining the last consent timestamp for audit context.
- Request bodies containing another `user_id` do not update another user's consent.
- Admin user edit path does not enable another user's telemetry consent even if the request body includes `tier2_telemetry_consent`.
